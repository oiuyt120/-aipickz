Continue.dev Review: I Tried the Open-Source Coding Assistant. It's Not Cursor. That's the Point.

Last month I had two editors open. Cursor on the left monitor, VS Code on the right. I was building a React component library and Cursor's composer kept suggesting patterns from a deprecated version. I couldn't switch models without restarting. So I'd code in VS Code and tab over to Cursor only when I needed AI help.

Dumb workflow. I knew it. But I couldn't find a tool that did what I wanted: AI assistance inside my editor, with my choice of model.

Then I found Continue.dev.

What it does

Continue is an open-source AI code assistant. It runs as a VS Code extension and a JetBrains plugin. It does not replace your editor. It sits inside it. You highlight code, press Cmd+I, and chat with any model you want: Claude, GPT, local models through Ollama. Your choice.

Marcus, a backend engineer at a B2B SaaS company in Berlin, put it to me this way: "Every three months a new AI editor launches and I'm supposed to migrate my keybindings again. I wanted something that works in the editor I already spent two years tuning."

The setup

Install the extension, configure a model provider, done. I pointed it at my local Ollama instance running Llama 3 70B. It worked on the first try.

But "it works" and "it works well" are not the same thing.

The tab autocomplete is slower than Cursor's. The first suggestion of every session takes a beat. The model needs to warm up. Subsequent suggestions are fine, but that initial delay sticks in your head.

Priya, a full-stack developer in London who reviews code for a team of twelve, told me she disabled autocomplete entirely. "The inline suggestions were more distracting than helpful. But the chat is genuinely better than Cursor's because I can switch models per task. Claude for architecture. A local model for simple snippets."

Where it shines

The chat is Continue's real strength. Cmd+L opens a sidebar that understands your project. I asked it to explain a complex useEffect chain I had written. It pulled the relevant files, traced the data flow, and pointed out a stale closure I had missed.

Cursor's chat can do this too. The difference: I swapped from GPT-4 to Claude mid-conversation. Continue kept the thread alive across model switches. No context lost. That matters when you are deep in a debugging session and want a second opinion from a different model.

The /edit command is the other standout. Highlight a block, type "make this async with proper error handling," and it rewrites the selection in place. No copy-paste. No new file. It just replaces the code.

I also discovered something I did not expect: I started using the chat for questions that had nothing to do with the file I was editing. "How does Python's asyncio.gather handle exceptions?" I was in a Java file. Continue answered without complaint. Cursor would have needed me to open a new chat.

Where it falls short

Continue is not an editor. It does not have Cursor's agent mode, the AI that creates files, runs terminal commands, and debugs end to end. Continue can propose changes, but you apply them manually. That makes it slower for multi-file refactors.

It also does not remember your style. Cursor builds a model of your codebase over time. It learns your patterns. Continue treats each session separately. You can give it context files, but it won't develop a persistent understanding of your preferences.

And the autocomplete is configurable but not adaptive. Cursor adjusts suggestions to what you are typing in real time. Continue uses a fixed configuration. Once set, it stays set.

The verdict

I keep both editors open now. Cursor for heavy lifting: multi-file refactors, complex debugging, anything that needs agentic behavior. Continue for everything else: explaining code, inline edits, quick questions with whatever model I feel like using.

Marcus ended up with the same setup. "Cursor for building features. Continue for understanding them. I use Continue more often than I expected."

Continue.dev will not replace Cursor. But that is not the goal. If you want an AI that builds features for you, use Cursor. If you want AI help that works inside the editor you already know, with full control over your models and privacy, try Continue. It might not become your primary tool. But you will reach for it more than you expect.
